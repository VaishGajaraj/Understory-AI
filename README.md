# Understory

**An open benchmark for detecting ground disturbance beneath forest canopy using free NISAR L-band radar.**

Most forest monitoring can't see under trees. Optical satellites are blocked by cloud and canopy; short-wavelength radar scatters off the treetops. NISAR's L-band radar penetrates the canopy and returns from the ground — and repeat-pass interferometric **coherence change detection** can reveal where the ground physically changed between two passes, even under closed canopy: logging skid trails, access roads, mining pits.

This project is scoped as a question, not a platform:

> **Can NISAR L-band coherence detect documented forest degradation events — how early, at what minimum event size, and at what false-alarm rate?**

Nobody knows the answer on real NISAR data yet. We intend to publish it, favorable or not.

## Repository layout

This is a monorepo. Python packages carry the science and pipeline (the MVP); TypeScript carries the viewer and any web-facing layers.

| Path | Language | What it is |
|---|---|---|
| [`packages/understory-core`](packages/understory-core) | Python | Data plumbing: NISAR granule discovery/retrieval (ASF), coherence stack construction, tiling, caching. Application-agnostic. |
| [`packages/understory-detect`](packages/understory-detect) | Python | The science: seasonal coherence baselines, anomaly detection, persistence/geometry filters, and the scoring harness. Detectors are pluggable. |
| [`packages/understory-labels`](packages/understory-labels) | Data + Python | The labeled disturbance-event library: versioned schema, contribution path, validation tooling. CC-BY 4.0. |
| [`benchmarks/`](benchmarks) | Config | Benchmark configurations per geography (Amazon/Pará first). Each runs end-to-end from a single command. |
| [`apps/viewer`](apps/viewer) | TypeScript | Thin web viewer for alerts and benchmark reports. Added early only as scaffolding; grows when a real user exists. |
| [`docs/METHODOLOGY.md`](docs/METHODOLOGY.md) | — | The versioned methodology document. For a benchmark project, the method is the citable artifact. |

## Quickstart

Python (requires [uv](https://docs.astral.sh/uv/)):

```bash
uv sync                      # install all workspace packages + dev deps
uv run pytest                # run all Python tests
uv run ruff check .          # lint
```

TypeScript (requires [pnpm](https://pnpm.io/)):

```bash
pnpm install
pnpm -r build
pnpm -r test
```

Run the toy benchmark (checked-in miniature data, no credentials needed):

```bash
uv run understory-bench benchmarks/toy/config.yaml
```

Running a real benchmark requires NASA Earthdata credentials for ASF data access — see [`docs/DATA_ACCESS.md`](docs/DATA_ACCESS.md).

## Design principles

- **Benchmark first.** Precision/recall against external ground truth, detection lead over optical alerts, minimum detectable event size, and an honest failure analysis. Published tables are generated by the scoring harness, never hand-assembled.
- **Legible before clever.** v0 is statistical baselines plus persistence, clustering, and geometry filters. ML enters at v1 only if the benchmark shows it beats the filters.
- **No application layer without a named user. No feature in core that only one application needs.**
- **Open data outlives pipelines.** The label library is versioned and governed separately from code.

## Success and kill criteria

Stated before building: the signal is worth pursuing if precision against external ground truth reaches ~70% at operationally useful recall, events ≤ ~2 ha are detectable, and median detection lead over optical alerts in cloudy-season tropics is ≥ 3 weeks. If the numbers land short, the failure analysis gets published anyway.
